http client 连接池源码分析

apache http client对http的连接做了池化管理，让客户端可以复用http连接提高http请求性能。

http的连接池主要需要解决以下几个问题：

1.连接的创建

2.连接的回收

3.测试连接是否可用

日志打印打开debug模式，执行testOneThreadConnection方法时可以看到debug日志中

http-outgoing-0为编号0的连接，所有的请求都是用这一个连接，说明http请求复用了连接。接下来分析相关源码。

发起请求时调用的是 org.apache.http.impl.client.CloseableHttpClient#execute(org.apache.http.client.methods.HttpUriRequest) 方法，直到org.apache.http.impl.conn.PoolingHttpClientConnectionManager#requestConnection方法获取一个连接

这里的pool是连接池对象，里面记录了可用的连接，已租借的连接

org.apache.http.pool.AbstractConnPool#lease(T, java.lang.Object, org.apache.http.concurrent.FutureCallback)方法返回一个Future对象，主要实现逻辑在Future的get方法里面

再看org.apache.http.pool.AbstractConnPool#getPoolEntryBlocking方法

连接的回收在方法org.apache.http.impl.conn.PoolingHttpClientConnectionManager#releaseConnection中，将可用的连接重新加入到pool中

判断连接是否有效主要是调用org.apache.http.impl.BHttpConnectionBase#isStale方法。该方法用于检测连接是否可用，当网络连接长时间没有活动，可能会被关闭，但连接池并没有发现该情况，如果此时进行读取操作，就会抛出 IOException。因此，该方法用于尝试检测连接是否仍然可用，实现方式是在连接上进行一个非常短暂的读取操作，并在极短的超时时间内返回结果。
